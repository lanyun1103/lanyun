JAVA WEB

一般用于动态页面

JSP（服务端脚本语言）

服务器架构

CS：Client Server

客户端

缺点：
  1. 需要更新则必须全部更新
  2. 任何一个坏了都要维护
  3. 每一个客户端 都要安装客户端软件
BS：Broswer Server

通过浏览器直接访问服务器（tomcat）

JSP基于BS

tomcat：

重要的几个目录：
  • lib：tomcat的jar包目录
  • webapps：执行项目目录
  • work：由jsp翻译成的java和class目录
  • temp：临时文件
  • conf：配置文件 ​ server.xml（可设置各种参数，例如端口）

默认访问webapps里面的root文件夹，可以修改 访问的主页默认为文件夹里面的index.html，可在WEB-INF的xml文件中修改

网站文件结构：

WEB-INF：web.xml 也可以加入项目专有的lib/classes

JSP：在html中嵌套的java代码

虚拟主机：

使服务器访问一个域名时不通过解析IP地址，而是通过本地的hosts文件映射直接返回服务器IP地址

1.在server.xml中添加

<Engine name = "Catalina" defaultHost="www.test.com">
<Host appBase = "D:***" name = "www.test.com"><Context docBase = "D:***" path = "/"
                                                       </Host>
</Engine>

2.本机hosts文件添加映射

JSP运行流程：

IE请求tomcat服务器，服务器返回响应（处理下面第一次访问的步骤后，打印显示文字）

第一次访问：jsp->java（servlet文件）->class

第二次访问：不需要编译，直接访问class（如果没有改变的话）（改变的话会重新编译

小插曲：Intellij创建java web项目步骤
  1. 创建JAVA EE中的Web Application
  2. 创建完成add configuration添加本地tomcat，
  3. 添加atricated，
  4. 修改context

IDE中实战：

WEB-INF中的文件无法直接通过url访问，需要请求转发使用

如果想要项目可以同时使用servlet和jsp，需要添加tomcat/lib/servlet-api.jar（对于intellij来说应该是废话）

需要统一编码，推荐使用UTF-8

JSP基础：

JSP的页面元素：
脚本Scriptlet：
<%
    %>部分变量、java语句
<%!
    %>全局变量、定义方法
<%=输出表达式%>
<%="hello"+name%>输出
指令：
<%@ page ....%>

page：指定的属性

language：jsp使用的脚本语言

import：导入类

pageEncoding：jsp文件本身编码 jsp->java

contentType：浏览器解析jsp代码（一般用UTF-8统一）

注释：

html 可以被看到源码

java /*...*/

jsp <%– –%>

JSP内置对象（九个）：
  • out：输出内容
  • pageContext
  • request：请求，存储“客户端向服务端发送的请求”（包含状态，动作）
    • String getParameter(String name)：根据key返回value
    • String[] getParameter(String name)：根据key返回多个value
    • void setCharacterEncoding(“编码格式”)：设置请求编码
    • getRequestDispatcher(b.jsp).forward(request,response)：请求转发的方式跳转页面
    • getServerContext()：获取项目的ServletContext对象
  • response
  • session
  • application
  • config
  • page
  • exception
